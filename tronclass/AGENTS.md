# Agent 工作说明

## 目标
自动下载待处理作业附件，并基于附件要求完成任务。

## 必须执行的步骤
1. 在仓库根目录执行下载脚本：`python3 scripts/download_pending_attachments.py`
2. 附件默认下载到 `attachments/<course_id>/<activity_id>/`，先遍历所有新下载附件再开始作答。
3. 对每个附件提取任务要求并形成可执行清单（交付格式、截止时间、字数/结构、是否需要代码或文档）。
4. 完成附件要求的内容，并将结果保存为清晰命名的输出文件（按课程和活动分目录）。
5. 执行 `scripts/` 下脚本前，先检查仓库根目录 `.env` 是否存在且包含所需配置。
6. 若 `.env` 缺失或关键凭据为空，必须调用 `agent-browser` 发起登录流程，向用户询问登录名和密码并完成登录。
7. 登录成功后将可用凭据保存到 `.env`，再继续执行后续脚本与任务。

## PDF / 扫描件处理
- 如果 PDF 文本不可复制、内容为空白提取结果或明显为扫描件，必须启用 OCR 后再解析内容。
- 优先使用可用的 OCR 工具链（如 `pdftotext`、`tesseract`、`ocrmypdf` 或等效方案）获取可读文本。
- OCR 后需要抽样核对关键字段（题目、限制条件、提交要求）避免识别错误。

## 执行约束
- 未完成附件解析前，不要直接生成最终答案。
- 信息不完整时，先在输出中明确“缺失信息”并给出最小可行假设，再继续完成可完成部分。
- 保留关键中间产物：下载文件、OCR 文本、最终答案文件，便于复核。
- 凭据写入 `.env` 后应立即限制文件权限为仅当前用户可读写（如 `chmod 600 .env`）。
- 任务中如包含必须通过 GUI 完成的步骤，代理不执行该步骤；在提交文档中明确标注该步骤需用户手动操作，并预留空位。
- 针对代码题目，提交文档中直接给出代码块；除文档外，还必须在实际目录中保留可运行/可复核的代码文件。
- 如果提交文档中存在任何未完成步骤，输出文件名必须包含“_UNFINISHED”标识（或等效清晰标识）。
- 最终提交文档必须保持人工写作风格，不包含任何可识别为 LLM 生成过程的描述、免责声明或提示语。
